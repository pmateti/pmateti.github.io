Home work in this course is done individually, without help from others. The homework will be given piece meal and it is expected that you will work on them as and when given without waiting for a due date. The due date is announced on the course home page.

  1. (25 points) [Literate Programming] Reconsider the OO revision (yours or someone else's with their permission) of the C++ Wiggly Worms of Homework1. Express it as a literate program (LP). If you wish, (i) start from scratch, use your own design, but with essentially the same visual effect and functionality, (ii) rewrite it in a different language of your choice (Java, Python, ...). However, the LP version should exemplify the OOP design by contract principles. Recall that an LP author presents it in an order that suits the explanation of the program to fellow software engineers, not necessarily in the order that a compiler may expect it.

    You may use any of the LP tools: noweb, nuweb, cweb, intent, lep (Eclipse plugin), nano-lp, ...; some of these are ready to apt-get install packages in Debian/Ubuntu distros. Visit http://www.literateprogramming.com/ for further details. Professor Emeritus Donald Knuth, "fastest pen in the West", Turing Award Winner 199x TBD, developer of the TeX typesetting system, has several example programs to read. His "Programming Pearls, Literate Programming: The Common Words Problem", PDF, 1986 was a Required Reading.

  2. (25 points) [JML] This part of the home work is focussed on gaining experience with the use of JML by practicing on the source code of the Game of Life that you studied in Homework1. Thoroughly document at least two files, non-trivial, and intersting-to-you, using JML assertions (i.e., invariant, requires, ensures, assignable, ...). Recall that we want the weakest requires, and the strongest ensures and invariants.

    Use a JML Plugin (in Eclipse, or Intellij Idea) or a command line JML tool to runtime assertion check (RAC) the Game of Life. There are several JML plugins that you can discover through a web search; e.g., http://jmlspecs.sourceforge.net/openjml-updatesite It is useful to this class if you all try different JML plugins and report on your experience.

  3. (25 points) [JUnit or TestNG] This part of the home work is focussed on systematic testing. You are welcome to use any two jEdit class files, non-trivial, and intersting-to-you, as test targets. Members of a team must of course select different jEdit classes. You are welcome to include the results of this Homework2 in the Testing and Debugging Report of the course project.

    Use a JUnit Plugin or a TestNG plugin (in Eclipse, or Intellij Idea). The latest versions of these IDE already have these test plugins installed.

  4. (25 points) [FindBugs Plugin] This part of the home work is focussed on gaining experience with the use of a static analysis of source code tool named FindBugs. See also, http://plugins.jetbrains.com/plugin/?id=3847. You are welcome to use any two jEdit class files, non-trivial, and intersting-to-you, as targets. Examine closely the reports produced by the FindBugs tool. Which of the items do you agree with? How did you resolve it?
  5. (00 points) Please maintain a journal for this homework. Submit a summary.
