CS 7140 Advanced Software Engineering

Midterm • 100 points

This exam permits the use of a Linux/Windows laptop/PC running g++, javac, ideaj and other software development tools. It is otherwise a traditional closed book, closed notes exam. You are expected to be aware of what constitutes cheating, and expected to be honor bound. In particular, you are not permitted access to Internet (other than for Pilot submission) or any content already existing. Your answers are expected to be in your own words after digesting the relevant educational materials. Do not give or take help from others.

Submit your answers.pdf on Pilot.

  1. (5 points each) The following statements may or may not be (fully or partially) valid. Explain the underlined technical term occurring in each statement. Explain/ discuss/ dispute the statement. It is possible to write no more than, say, five, lines each, and yet receive full score.
    1. If specs are given only in English, it is hard to say whether the program is in error or requirements are.
    2. If the pre-condition of a piece of code S is true, you are asserting that S does terminate.
    3. The Design by contract principle requires the use of assertions.
    4. There are wicked problems for which we cannot write programs.
    5. A functional specification is a rigorous description of the relationship between the inputs and outputs.
  2. (15 points each)
    1. Consider an example implementation of Quick Sort in Java. Give its weakest pre-condition and strongest post-conditions and strongest loop invariant for the outermost loop of the partition method.
    2. Develop clearly stated specs, in well chosen English, Math, Logic, for Naur's Telegram problem reproduced here. Resolve all ambiguities, "sins", etc. sympathetically, and focus on specs. "Given a text consisting of words separated by BLANKS or by NL (new line) characters, convert it to a line-by-line form in accordance with the following rules: (1) line breaks must be made only where the given text has BLANK or NL; (2) each line is filled as far as possible, as long as (3) no line will contain more than MAXPOS characters."
    3. Develop a design for the above in VHLL, a very high level pseudo code language. The pseudo code should be good enough for a sophomore to code it in Java without any interaction with you.
    4. Describe the Liskov Substitution Principle rigorously. [Yet to cover this, Summer 2018]
    5. Explain each line of the JML assertions shown in group 5 of jml-one-page-ex-1.pdf [Ignore JML specifics.]
