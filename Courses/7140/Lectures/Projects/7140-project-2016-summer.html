1.1 Expectations of our PDF-project

  1. Start with (or build afresh) a pdf-viewer-editor that can
    1. Read "real world PDF documents" (that may slightly deviate from the PDF-spec).
    2. Split a PDF file into several, e.g., by page.
    3. Extract text (Accuracy is important).
    4. Edit Annotations.
    5. Search for words and position a cursor.
  2. Write the software engineering documents as outlined in Project Work.
  3. Focus on the chosen library (one of iText, jPod, …), and develop assertions. Revise the code applying good smells, and refactoring, discussed in the course.

1.2 Non-Expectations

  1. Not expected: Edit a pdf file as if editing a MSWord document. E.g., not expected to be available: replace one word with another, insert/ delete a word
  2. Not expected: Accept input from a stylus/pen. Handwriting. Freehand drawing.
  3. Rendering – leave it alone.

1.3 A Target

  1. http://www.tracker-software.com/product/pdf-xchange-editor Try this freeware. Can we duplicate the functionality of this? Even if we cannot compete with its speed.
