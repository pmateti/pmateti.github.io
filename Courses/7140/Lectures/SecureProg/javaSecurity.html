2 The Security Manager

  1. A security manager is an object.
  2. Defines a security policy for an application.
  3. system policy file ${JAVA_HOME}/lib/security/java.policy
    1. e.g ., /usr/lib/jvm/jdk1.7.0_13/jre/lib/security/java.policy
    2. http://docs.oracle.com/javase/tutorial/security/userperm/policy.html
  4. Java library checks with SM whenever a "dangerous" operation is about to be called.
  5. SM can raise a SecurityException

3 Java Classloader

  1. Part of the JRE that dynamically loads classes into the JVM.
  2. The bootstrap class loader loads the core libraries {$JAVA_HOME/jre/lib}
  3. The extensions class loader loads the code in the extensions directories {$JAVA_HOME/jre/lib/ext}
  4. The system class loader loads code found on $CLASSPATH.
  5. Classes loaded by different loaders do not have package-private access to one another even if they have the same package name.
  6. http://en.wikipedia.org/wiki/Java_Classloader Reference
  7. http://www.onjava.com/pub/a/onjava/2005/01/26/classloading.html Reference
  8. http://incubator.apache.org/depot/version/jar-hell.html Reference

4 Java Sandbox

  1. The sandbox is a set of rules. Similar to "jail", chroot.
  2. Byte Code Verifier – Checks ("verifies") JVM code before allowing to run.
  3. Applet Class Loader – when and how an applet can add/replace classes to JRE. Applets are sent by a web server.
