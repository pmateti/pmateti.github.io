Software Architecture: OOP v Functional Prog by Meyer

Bertrand-Meyer-Arch-OO-Func-2009-Orig.pdf meyer-pudding-ex.png Bertrand-Meyer-Arch-OO-Func-2009.pdf Bertrand-Meyer-OOP-v-Func-Split

  1. A toy example: From (precise) descriptions of puddings, compute
    1. sugar content
    2. estimate time to make
    3. retrieve instructions to make it
  2. One Approach
    1. List all puddings (Trifle, lemon upside-down pudding, Dutch apple cake, Christpudding)
    2. For each pudding, write down sugar content, time to make, instructions, etc.
  3. Another Approach
    1. Define a small set of 'pudding combinators.'
    2. Define all puddings in terms of these combinators.
    3. Calculate sugar content from these combinators too.
    4. An Example of a Pudding:
  4. Descriptions Of Puddings
    1. salad = on-top-of topping main-part
    2. topping = whipped (take pint cream)
    3. main-part = mixture apple-part orange-part
    4. apple-part = chopped (take 3 apples)
    5. orange-part = optional (take 6 oranges)

0.1 Compute S (i.e., Sugar Content): Functional Prog Solution

  1. S (on-top-of p1 p2) = S (p1) + S (p2)
  2. S (whipped p) = S (p)
  3. S (take q i) = = q * S(i)
  4. …

0.2 Advantages

  1. When we define a new pudding/recipe, we can calculate its sugar content with no further work.
  2. Only if we add new combinators or new ingredients would we need to enhance S.

0.4 Functional Programming Version

  1. Europian Option: european t u = get (truncate t (or u zero))
  2. H below denotes the expiry date, or horizon
  3. H (zero) = infinity , where infinity is a special value with the expected properties
  4. H (or c1 c2) = max (H (c1), H (c2))
  5. H (anytime c) = H (c)
  6. H (truncate t c) = min (t, H (c))
  7. H (get c) = H (c)
