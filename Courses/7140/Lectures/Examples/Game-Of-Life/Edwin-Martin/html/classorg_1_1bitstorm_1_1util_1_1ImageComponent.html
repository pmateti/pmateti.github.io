org.bitstorm.util.ImageComponent.ImageComponent ( Image  image)

Constucts a ImageComponent.

This constructor uses the MediaTracker to get the width and height of the image, so the constructor has to wait for the image to load. This is not a good idea when getting images over a slow network. There is a timeout of 3 sec.

Maybe one time the MediaTracker can be replaced, so it returns when the width and height are known, not when the whole image is loaded. Then the constructor will be much faster and better suited for loading images over a slow connection.
