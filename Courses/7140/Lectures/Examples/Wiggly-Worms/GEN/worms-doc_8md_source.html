15 <li>We are using the word worm as in an earth worm. Our worms travel
16 on a rectangular (x,y)-grid plane. They are born with their head
17 positioned just below this plane (i.e., their body hangs on z-axis
18 vertically down). We have three kinds of worms: vegetarian (eat only
19 carrots), cannibal (eat everything), scissorhead (cut the others).

86 is at the end just to show that an array field does not have to be the

115 This illustrates a good use of macros. xworms[] is an array so

123 <li>
124 <pre>
125 typedef struct { /* A Square On the Ground */
126  char onec; /* one character */
127  char attr; /* its color */
128 } ASOG;
129 
130 ASOG passive[MAXrow * MAXcol]; /* for Carrots and Dead Worms */
131 ASOG screenImage[MAXrow * MAXcol]; /* what gets displayed */
132 #define asog(ptr, row, col) (ptr + (row - 1)*screenCols + col - 1)
133 </pre>
134 <p>
135 These two (passive and screenImage) together is what you see on the
136 monitor. Note that they are 1-D arrays. The macro takes
137 what in 2-D-array-index notation would be a[i][j] and
138 turns it into a 1-D ptr-based left-value.
139 
140 <p>
141 Why -1? Because, the screen coordinates start from 1 not 0, but C
142 array indices begin at 0.
143 
144 <p>
145 Why are we not multiplying with MAXcol in
146 <tt>
147 #define asog(ptr, row, col) (ptr + (row - 1)*screenCols + col - 1)
148 </tt>
149 
150 Because, even though we declared the two 1-D arrays passive and
151 screenImage to be sufficiently long to be considered MAXrow-by-MAXcol
152 2-D arrays, we actually are using them as screenRows-by-screenCols
153 arrays.
154 
155 <li>
156 We display the carrots and worms by first copying the entire passive[]
157 into screenImage[]. Then we draw the images of the live worms
158 onto screenImage[]. We then blast the entire array screenImage to
159 the monitor screen via the puttext().

189 <pre>
190 int isHungry(WORM * wp)
191 {
192  int m = wp->stomach;
193  int n = wp->nsegs * worm_info[wp->type].capacity;
194 
195  return (wp->status == ALIVE && 4 * m < 3 * n);
196 }

198 <p>
199 Of course, it does not make sense that dead worms are hungry. Then
200 why are we testing for it? To make this routine more robust. When we
201 make a worm die, we simply change its status. We do not change its
202 stomach value. So, just in case you call this routine with such a
203 way ...
