SmallSet Requirements

1 Abstract

This note describes the req/ spec/ design of a (small/ fixed size) set of integers. This is expected to be a good example of software engineering design "in the tiny". There is also a link to a Java implementation.

2 SmallSet Requirements

  1. A set is a concept from discrete mathematics.
  2. "smallset": No bigger than, say, 1000 elements.
  3. Insertion "fails" beyond 1000.
  4. This SmallSet is NOT the same as Java Set container.

2.1 SmallSet Operations: Names

  1. For now, just the names
    1. construct-empty-set (or cons)
    2. insert-an-item, delete-an-item
    3. item-is-in-the-set
    4. number-of-items-in-the-set
    5. union-with, diff-with
  2. All the above should be well-specified
  3. ignored detail: item ==> int
  4. all the above converges to "API"

2.2 SmallSet Operations: Signatures

  1. insert-an-item: Set, Item –> Set
  2. delete-an-item: Set, Item –> Set
  3. item-is-in-the-set: Set, Item –> Boolean
  4. number-of-items-in-the-set: Set –> Integer
  5. union-with: Set, Set –> Set
  6. diff-with: Set, Set –> Set
  7. construct-empty-set: –> Set

2.3 SmallSet Requirements Clarifications

  1. Insertions are far more frequent cf to other ops
  2. item-is-in-the-set(S, e): searched for again often

3 SmallSet Operations: Semantics

  1. Should we describe semantics here in the requirements?
  2. We decided to just depend on what discrete math tells us.
