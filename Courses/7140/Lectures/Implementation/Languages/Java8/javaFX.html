1 Abstract

JavaFX is the GUI building mechanism for Java applications. It replaces Swing. Other alternatives are libGDX and Unity. JavaFX GUI uses threads, lambda expressions, etc.

  1. Container for a scene graph.
  2. A stage can have multiple scenes.

  1. Organizer of nodes in scene graph

  1. Tree of nodes; there is a root node.
  2. A branch may have children (e.g., Group, Region)
  3. Leaf does not have children, (e.g., Rectangle, Circle)

6 JavaFX Games

  1. Translation: Shifts a node from one position to another
  2. Rotation: Rotates a node around some point

6.1 The Game Loop

  1. Classically, a game will have at its heart a loop like this:
  2. while (notEndOfGame) { updateGameState(); renderGame(); }
  3. The frequency of the loop is referred to as the frame rate, and can determine the “smoothness” of the game play. 35mm film runs at 24FPS. Video games often run at about 30-60FPS.

6.2 A Game Loop in JavaFX

  1. JavaFX takes care of rendering, and can update at about 60FPS when possible.
  2. The AnimationTimer class can be used to get a call back every frame. 
new AnimationTimer() {
   @Override
   public void handle(long now) { updateGameState(now); }
}.start();
