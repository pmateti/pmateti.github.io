  1. (35; 5 points each) The following statements may or may not be (fully or partially) valid.   Explain the technical terms occurring in the statement.   Explain/ discuss/ dispute the statement.  It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. A host has two NICs each on a different network.  It is possible, for such a machine, to establish a fully capable routing table using no more than two rows.
    2. It is possible to setup a Linux/ Unix system without a single suid program.
    3. If the MD5 sums of two files are equal, this does not necessarily mean that the files are equal in content.
    4. Masquerading, spoofing, and smurfing are related terms.
    5. Backdoors are used to install rootkits.
    6. In a TCP segment with SYN=1, the SEQ number can be non-zero.
    7. The terms "proper configuration" versus "fortification" versus "hardening" differ in the intensity of the work required of the system administrator.

     

  2. (4*10 points)
    1. Explain how public-key encryption scheme can be useful in user authentication.
    2. Suppose that an attacker has acquired privileges to read/write/execute any file on a Unix system.   Suppose his/her goal is to obtain one hundred userid-password combinations, and replace what ever changes he/she may have made with their originals so that this activity has a greater chance of going unnoticed.  Describe what files are changed where and how.
    3. What are one-time passwords?
    4. Explain in detail why we should use ssh and sftp instead of telnet and ftp?
