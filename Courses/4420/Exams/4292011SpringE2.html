  1. (8*5 points each) The following statements may or may not be (fully or partially) valid.  First, explain the term underlined in the statement.  Then, explain/ discuss/ dispute the statement.  It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. A "script kiddie" is a synonym for "hacker".
    2. Hackers, as distinct from attackers, do have ethics.
    3. Probing is used to detect the weaknesses of a target machine.
    4. Address Space Layout Randomization (ASLR) helps prevent backdoors.
    5. A firewall can be configured to prevent IP spoofing.
    6. An IP filtering package must be run on a router.  It has no effect if run on a normal host with a single network card.
    7. WEP of 802.11 uses SSID as the shared key.
    8. For something to be called a worm, it must embed in such a way that it is executed only intermittently.
  2. (10*5 points)
    1. (10 points) Describe how/why stack top randomization makes buffer overflow exploits difficult.
    2. (10 points) Suppose that an attacker has acquired root privileges on a Linux system.   Suppose his/her primary goal is to obtain one hundred userid-password combinations.  A secondary goal is to do this without being noticed.  Describe specifically what files are changed, and perhaps later replaced with originals, where and how.
    3. (10 points) Describe the functionality of any five security related entries in /proc/sys/kernel
    4. (10 points) Describe any two of the Principles of Robust Programming of Matt Bishop's article.
    5. (2*5 points) Consider the function shown below.  It reads a file name from the standard input and returns its protection mode. 
      short int protectionMode(void)
{	struct stat stbuf;
	char inbuf[100];
	gets(&inbuf);
	stat(inbuf, &stbuf);
	return stbuf.st_mode & 0777;
}
      1. Describe how to exploit its features.
      2. Write its pre- and post-conditions.
