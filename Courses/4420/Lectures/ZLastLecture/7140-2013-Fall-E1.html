CS 7140 Advanced Software Engineering

2013 Fall • Midterm • 100 points

This take-home exam permits the use of a Linux/Windows laptop/PC running javac, g++, Eclipse, Idea, and other software development and drawing tools. It is otherwise a traditional closed book, closed notes exam. In particular, you are honor bound *not* to Internet surf or access any content already existing (other than as indicated) once you access the midterm until you turnin the answers. The primary reasons in making this a take-home are to relieve time pressure and give you a comfortable (computing/home) environment. Do not give or take help from others.

Submit your answers by Sat Nov 02, 2013 12:00 noon on gandalf.cs.wright.edu using ~pmateti/cs7140/turnin MT answers.pdf

  1. (5 points each) The following statements may or may not be (fully or partially) valid. Explain the underlined technical terms occurring in each statement. Explain/ discuss/ dispute the statement. It is possible to write no more than, say, five, lines each, and yet receive full score.
    1. Suppose we wrote down specs rigorously/ formally using mathematics and logic, and requirements in carefully worded English. There is no method other than a judge+jury system to decide if these two are consistent with each other.
    2. If the pre-condition of a piece of code S is false, we are asserting that S does not terminate and therefore we can have fun and claim a preposterous post-condition.
    3. The Design by contract principle is a variation of the Liskov Substitution Principle.
    4. Once we write a program for a wicked problem it is no longer wicked.
    5. Suppose we are revising a 1970s program, that used as little memory as possible, into a modern one, that uses memory liberally, in the hope of significantly improved performance. We can use the old program as a functional specification for the new one.
  2. (15 points each)
    1. Consider our Wiggly Worms program. We can intuitively describe the "connectedness" of a worm's body as "we can draw a line possibly with a few turns without lifting our pen starting from the head body segment to the tail". State this as an assertion using mathematical notations using the typedefs originally given or the class definitions you have derived from it.
    2. Here is a link to BinarySearch.java. Focus on the public static int rank(int key, int[] a) method. Give, in JML, its weakest pre-condition and strongest post-condition and strongest loop invariant for the while loop of the method. You are welcome to use JML plugin in Eclipse or IDEA.
    3. Our requirements for the SmallSet of Integers have changed as follows. (i) It is no longer a set but a bag -- we now want a SmallBag of Integers; (ii) No bigger than 1000 elements; unchanged (iii) Deletions (as well as insertions) are frequent ops (iv) item-is-in-the-set(S, e) is now item-is-in-the-bag(S, e) and furthermore not used often (iv) Some ops can "fail", but only if they must. Modify our SmallSet design so that it is now a SmallBag design. Present it in a VHLL. You will lose points if it is almost Java.
    4. Give an example OOP program of your own in Java or C++ that illustrates the Liskov Substitution Principle.
    5. A specific hash trie in the context of Common Words Problem is shown in cwp-hash-trie.pdf. We now insert the word "bend". Show the resulting hash trie.
