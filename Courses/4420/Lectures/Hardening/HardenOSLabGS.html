Acceptable Lab Report structure 10  
Download a recent (stable or unstable, if not *the* latest, choose within the last 6 months) bundle of source code of Linux from https://www.kernel.org/ pub/ linux/ kernel/ v3.0/ Unpack it and establish a symbolic link named linux to it. Show the results of disk usage, du at this instant of the linux dir. 5  
In the following steps, you are welcome to follow some other procedure to build the kernel, etc., but you must explicitly describe and cite your source of info. 00  
cd linux; make xconfig Select any 5 or more components from the list of Building a Secure Linux Kernel. Include portion(s) of the generated .config file that shows your selections. 30  
Describe in detail your kernel config, the reasons for your choices and observations. 15  
Build the kernel and the modules: make bzImage modules Time this using the bash prefixes time and nice. Show the results of disk usage, du of the linux dir after the build. 10  
Locate and copy the bzImage, config and System.map files into LXU (Linux-on-USBD) appropriately. (initrd should not be needed.) Fix menu.lst (legacu Grub) or grub.cfg (Grub 2) and show that your machine can boot with the new kernel. You can do a uname -a; cat /proc/version after the successful boot and capture the stdout to show. 20  
Develop a quick test that demonstrates the enhanced security of the kernel. 10  
Bonus Points: Recompile the Linux kernel, but using a compiler other than gcc, using the same .config as above. Verify that LXU works properly when its original kernel is replaced by this one 20B  
Bonus Points:  Compile, using gcc, the Linux kernel that maximizes the use of modules. So, you will need initrd. Keep the security choices you made the same as before. Verify that LXU works properly when its original kernel and initrd are replaced by these. 20B
