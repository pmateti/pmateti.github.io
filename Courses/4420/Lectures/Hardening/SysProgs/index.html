2 System Programs

What is a system program? A consensus definition is it is a program without which the OS is (semi?) dysfunctional. E.g., the kernel itself can mount file volumes, but most of the mounts are done through /sbin/init through the mount program. A system program is run with the privileges of the root.

2.1 A Fairly Complete List

  3. most of /sbin, /usr/sbin (can be pruned heavily)
  4. all suid root programs. Non-suid-root sys programs must be invoked by root.

5 System Programming Example: chroot

A chroot dnm changes the official root directory, from whatever it was to dnm, for the process that invoked it and its children. To a program that is invoked in such a modified environment the (new) root is the path name argument dnm given. 

# cd location-of-new-root
# mount -t proc /proc proc/
# mount --rbind /sys sys/
# mount --rbind /dev dev/

Recommended Reading https://wiki.archlinux.org/index.php/change_root
