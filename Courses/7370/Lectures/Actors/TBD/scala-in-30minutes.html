5 Scala can be Hard to Learn

  1. Voluminous documentation aimed at programming newbies. Too many (incompatible) versions. 24,066 questions tagged on http://stackoverflow.com/questions/tagged/scala
  2. Functional Programming paradigm: Imagine no variable can be assigned more than once. Anonymous Functions. Currying.
  3. Lazy values.
  4. Futures, Promises
  5. Java classes/interfaces remixed: Multiple inheritance, Traits
  6. Type inheritance
  7. Omission of "things": the dot, the semicolon, …

7 References

  1. Martin Odersky, "Scala by Example". Highly recommended read Chapter 17: Abstractions for Concurrency (10+ pages). The full book is at http://www.scala-lang.org/docu/files/ScalaByExample.pdf, 150+ pages. Martin Odersky and others at EPFL, Switzerland are the designers of Scala. Apparently the pdf gets generated every day. Caution: "Scala by Example": Despite the name of the book, only a few are ready-to-compile-and-run examples.
  2. Michel Schinz, Philipp Haller, "A Scala Tutorial for Java programmers", http://www.scala-lang.org/docu/files/ScalaTutorial.pdf, 2014, 15 pages.
  3. https://typesafe.com/activator is the place for ready-to-compile-and-run examples of Scala (or Java) with Akka.
