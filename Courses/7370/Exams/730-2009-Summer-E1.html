  1. (6*5 points) Explain/ discuss/ dispute/ compute the following statements. It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. Consider the following program segment: 
      	  co <await x >= 3 -> x := x - 3>
	  || <await x >= 2 -> x := x - 2>
	  || <await x == 1 -> x := x + 5>
	  oc
      Let P be a predicate that characterizes the weakest deadlock-free precondition for the program, i.e., the largest set of states such that, if the program is begun in a state satisfying P, then it will terminate if scheduling is weakly-fair. Determine P. Explain how you arrived at P.
    2. Compute wp(S, i = 2), where S is if i < 2 -> i := i+1 [] i > 2 -> i := 2 fi
    3. The semantics of  S1 || S2 permits their execution as if it is S1;S2 True/False? Explain.
    4. The code V(s); P(s) is always equivalent to a no-op ("skip", or do-nothing). True/False? Explain.
    5. In the context of RPC, a procedure designed to be called remotely can use global variables to communicate with the caller. True/False? Explain.
    6. Marshalling in RPC is no more than a tight-packing of the bytes of the input arguments. True/False? Explain.
  2. (30 points) The Recursive Data Representation: Small Set of Integers of Hoare's CSP paper is reproduced below. (a) (20 points) Extend it to respond to a command S(1)!remove(x) from S(0) that removes x from the set, if it contains it, and does nothing if it does not. (b) (10 points) Include a convincing argument that your extension is correct. 
        S(i: 1 .. 100) ::
    *[   n: integer; S(i-1)?has(n) --> S(0)!false
     []  n: integer; S(i-1)?insert(n) -->
            *[  m: integer; S(i-1)?has(m) -->
                 [   m <=  n --> S(0)!(m = n)
                 []  m >   n --> S(i+1)!has(m)
                 ]
             [] m: integer; S(i-1)?insert(m) -->
                 [  m < n --> S(i+1)!insert(n); n := m
                 [] m = n -->  skip
                 [] m > n --> S(i+1)!insert(m)
     ] ] ]
  3. (40 points) Consider the following as an await-statement based model of the classic Dining Philosophers problem. The expression i + 1 throughout is modulo 5. The up and dn (down) refer to whether the philosopher picked the fork up or put the fork down.
    1. (20 points) Using the Pass-the-Baton technique systematically convert it to an equivalent algorithm using semaphores only. For now, disregard if the above is a ``solution'' or not.
    2. (10 points) Is this ``solution'' free from starvation? Explain your answers fully. A simple yes/no will not do.
    3. (10 points) Is this a ``solution'' to the Dining Philosophers problem? If so, explain. If not so, fix it at the await-statement level . 
       var up[0..4], dn[0..4] initially all 0;
   philosopher[i: 0..4]::
     do true ->
   	think
   	<await up[i] == dn[i] --> up[i] ++>
   	<await up[i+1] == dn[i+1] --> up[i+1] ++>
   	eat
   	<dn[i] ++>
   	<dn[i+1] ++>
     od
