This is a closed book, closed notes exam. Do not give or take help during the exam.

  1. (5*5 points) The following statements may or may not be (fully or partially) valid.   Explain/ discuss/ dispute the statement.  It is possible to write no more than, say, five thoughtful sentences each, and yet receive full score.

    1. Every distributed system must also be a concurrent system.
    2. It is not possible to have a deadlock among three processes if they are using exactly one semaphore.
    3. Asynchronous  message passing is more fundamental than synchronous message passing.

    4. The happened before relation, as in the book, requires a logical clock for every process.  But  a typical machine (node) in a distributed system runs several processes. It is sufficient to maintain a logical clock per node that is shared by all processes running on that node.

    5. Peer-to-peer distributed computing is just an academic term for the commercial term client-server computing.
       
  2. (5+5+5 points) It is possible to write no more than, say, five thoughtful sentences each, and yet receive full score.
    1. Discuss why it is difficult to implement a semaphore in the truly distributed context.
    2. Discuss if any solution to the distributed mutual exclusion problem be thought of as a solution to the problem of implementing semaphores in a distributed system, and vice versa.
    3. Describe the problem of "termination detection."
       
  3. (30 points) Consider the following as an await-statement based model of the classic Dining Philosophers problem.  The expression i + 1 throughout is modulo 5. The up and dn (down) refer to whether the philosopher picked the fork up or put the fork down. 
       var up[0..4], dn[0..4] initially all 0;
   philosopher[i: 0..4]::
     do true ->
   	think
   	<await up[i] == dn[i] --> up[i]++ >
   	<await up[i+1] == dn[i+1] --> up[i+1]++ >
   	eat
   	<dn[i]++ >
   	<dn[i+1]++ >
     od
    1. (20 points) Using the Pass-the-Baton technique systematically convert it to an equivalent algorithm using semaphores only. For now, disregard if the above is a ``solution'' or not.
    2. (10 points) Is this ``solution'' free from starvation? Explain your answers fully. A simple yes/no will not do.

     

  4. (10+20 points) You are given a large (in the zillions) set S of pairs of unsigned integers (x,  d).  Each pair represents a horizontal line segment of length d in the Euclidean plane whose left end point is at coordinate (x, y) with y = 0.   The segment  (x1, d1) contains (x2, d2) if  x1 ≤ x2 and x1 + d1 ≥ x2 + d2.  The set S is a compact representation of the set of points (a, 0) contained in some line segment (x, d) of S, i.e., x ≤ a ≤ x + d with a being a real number.  We wish to compute the set Smin which minimizes the size (== number of pairs) of S while representing the same set of points as S.

    1. (5 + 5 points) As usual, we wish to maximize concurrency, we prefer a symmetric solution, we rate correctness much higher than efficiency, and cannot tolerate deadlocks or livelocks. Defend why we  cannot tolerate deadlocks or livelocks (5 points), and why we rate correctness much higher than efficiency (5 points) .

    2. (10 + 10 points) Fully explain (10 points) your solution design  in English/pseudo-code for Linda and on a fresh page implement  (10 points) it in C-Linda. Assume that the tuple space already contains <"t", x, d> of S, and the size of this bag <"sz", N>. You lose 2 points for each use of inp or rdp.
