Summer 2004// Final Exam // 75 minutes // 100 points
Prabhaker Mateti

This is a closed book, closed notes exam, except for the few items listed in the last class. Do not give or take help during the exam.
  1. (5*5 points) The following statements may or may not be (fully or partially) valid.   Explain/ discuss/ dispute the statement.  It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. There are examples of safety properties that are liveness properties.
    2. Detection of termination is difficult only in peer-to-peer distributed computing not in client-server computing.
    3. Every client-server program is based on RPC (-like) mechanism.
    4. Token-passing algorithms were discussed under the heading of ``asynchronous message passing'' because token-passing does not make sense under synchronous message passing.
    5. The happened before relation, as discussed, requires a logical clock for every process.  But  a typical machine (node) in a distributed system runs several processes. It is sufficient to maintain a logical clock per node that is shared by all processes running on that node.
  2. (10+10+5 points) It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. Discuss why it is difficult to implement a semaphore in the truly distributed context.
    2. Discuss if all of the ACKs are necessary.
    3. Can any solution to the distributed mutual exclusion problem be thought of as a solution to the problem of implementing semaphores in a distributed system, and vice versa.
  3. (5*10 points) You are given a large (in the millions) bag of triplets of unsigned integers (x, y, d).  Each triplet represents a horizontal line segment of length d in the Euclidean plane whose left end point is at (x, y).   Two triplets (x1, y1, d1) and (x2, y2, d2) are said to be connected if  y1 == y2 and either x1 + d1 == x2 or x2 + d2 == x1.  A triplet (x1, y1, d1) contains (x2, y2, d2) if y1 == y2 and x1 <= x2 and x1 + d1 >= x2 + d2.  We would like to minimize the size (== number of triplets) of  the given bag of triplets so that every original triplet is contained in some triplet in the resulting collection.  Two solutions, one in CSP and another in C-Linda are to be developed.  It is not required that both use the same "idea."  Present your design and implementation separately, each with full explanations.  As usual, we wish to maximize concurrency, we prefer a symmetric solution, we rate correctness much higher than efficiency, and cannot tolerate deadlocks or livelocks.
    (a) Design  and (b) implement a solution in C-Linda. Assume that the tuple space already contains the triplets <"t", x, y, d>, and the size of this bag <"sz", N>. You lose 5 points for each use of inp or rdp.
    (c) Design and (d) implement a solution in CSP.  The bag is generated by process A whose structure is
    A :: *[ ... --> B!(x, y, d) ].  Develop the process B and any others.
    (e) Defend why we wish to maximize concurrency,  why we prefer a symmetric solution, and  why we rate correctness much higher than efficiency.
