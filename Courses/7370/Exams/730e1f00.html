  1. (10*5 points) Explain/Discuss/Dispute/Answer, in a few lines, the following.
    1. Discuss the semantics of the fat-bar operator as in S1 [] S2.
    2. The little-endian-ness of a machine has no relevance to RPC.
    3. What does svc_run() do? Why does it not ``return''?
    4. Explain what dispatching is, in the context of RPC.

    6. ``Every philosopher will eventually get hungry.'' Explain if this is a safety, or liveness property, or perhaps neither.
    7. Explain the difference in meaning of the notation {P} S {Q} versus wp(S, Q).
    8. The technique of  passing the baton can only be used for the Readers/Writers Problem, and not for mutual exclusion, starvation-free
      or not, of two or more processes.
    9. Any group of processes communicating only via message passing constitutes a ``distributed system.''
    10. Consider the following program segment:
      11. co <await x >= 3  -> x := x - 3>
      || <await x >= 2  -> x := x - 2>
      || <await x == 1  -> x := x + 5>
      oc

      Let P be a predicate that characterizes the weakest deadlock-free precondition for the program, i.e., the largest set of states such that, if the program is begun in a state satisfying P, then it will terminate if scheduling is weakly-fair. Determine P. Explain your answer.
