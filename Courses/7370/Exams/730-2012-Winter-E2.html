  1. (15*5 points) In the following, it is possible to write no more than, say, ten, sentences each, and yet receive full score.  Explain/ discuss/ dispute/ compute/ construct/ prove.
    1. "A distributed system is one in which the failure of a computer you didn't even know existed can render your own computer unusable." Is this a good definition?
    2. In a collection of semaphores that constitutes a split binary semaphore, at least one of them must be 1.
    3. A safety property is any property that is asserted to be true throughout the life of a process.
    4. Detection of termination is difficult only in peer-to-peer distributed computing not in client-server computing.
    5. Construct the ssend(B, msg) and srecv(B, msg) primitives of SMP that a process A uses in communicating with another process B from asend(B, msg) and arecv(B, msg) primitives of AMP. 
    6. Token-passing algorithms were discussed under the heading of AMP because token-passing does not make sense under SMP.
    7. The happened before relation, as discussed, requires a logical clock for every process.  But  a typical machine (node) in a distributed system runs several processes. It is sufficient to maintain a logical clock per node that is shared by all processes running on that node.
    8. Given a solution to the distributed mutual exclusion problem, we can construct semaphores in a distributed system, and vice versa.
    9. Reconstruct the binary "tree" serialized below. List each node as a triplet of (info integer, left-link and right-link), as usual.  The first digit in the sequence is the offset of the root node.
      marshalled sequence: 2 3 4 7 4 7 8 4 A 1 4 2 7
      offsets in hex:      1 2 3 4 5 6 7 8 9 A B C D
    10. Verify if the precondition and statement {x  ≥ 9 } < x := x - 9 > interferes with the triple
      {x is odd} < x := x + 2 > {x is even}.
    11. Let S contain < await B → T >. Let S' be S with the await statement replaced by do not B → skip od; T. Then, {P} S {Q} holds if and only if {P} S' {Q} holds.
    12. In SR, what is the effect of sending a message to an op implemented as a procedure?
    13. Explain, with an example, the map/reduce idea behind Hadoop.
    14. Consider the code block below. Do any of the temporal assertions shown hold? When? How?
      int x := 0; int y := 0;
      do < await true → x := x + 1>
      [] < await true → y := y + 1>
      od             		  always x = 0
      eventually x > 5
      eventually always y = 3
    15. We wish to maximize (a) concurrency, we prefer (b) symmetric solutions, we rate  (c) correctness much higher than efficiency, and cannot tolerate (d) deadlocks or (e) livelocks.  Defend why for each of these five.
