This is a closed book, closed notes exam, except for the few items listed in the last class. Do not give or take help during the exam.
  1. (15*5 points) In the following, it is possible to write no more than, say, ten, sentences each, and yet receive full score.  Explain/ discuss/ dispute/ compute/ construct/ prove.
    1. Define a split binary semaphore.
    2. There are examples of safety properties that are also liveness properties.
    3. Detection of termination is difficult only in peer-to-peer distributed computing not in client-server computing.
    4. Construct the ssend(B, msg) and srecv(B, msg) primitives of SMP that a process A uses in communicating with another process B from asend(B, msg) and arecv(B, msg) primitives of AMP. 
    5. Token-passing algorithms were discussed under the heading of AMP because token-passing does not make sense under SMP.
    6. The happened before relation, as discussed, requires a logical clock for every process.  But  a typical machine (node) in a distributed system runs several processes. It is sufficient to maintain a logical clock per node that is shared by all processes running on that node.
    7. Reconstruct the binary "tree" serialized below. List each node as a triplet of (info integer, left-link and right-link), as usual.  The first digit in the sequence is the offset of the root node.
      marshaled sequence: 2 1 4 2 6 3 4 9 4 0 2 9 9
      offsets in hex:     1 2 3 4 5 6 7 8 9 A B C D
    8. Can a random sequence of 13 digits be substituted for the marshaled sequence above and yet be able to construct a "tree"?
    9. Does the precondition and statement {x  ≥ 4 } < x := x - 4 > interfere with the triple
      {x is odd} < x := x + 5 > {x is even}.
    10. Using the technique of weakened assertions, prove that {x = 1} S {x = 7} is a theorem, where S is  co < x := x+1> || < x := x+2> || < x := x+3> oc
    11. Discuss why it is difficult to implement a semaphore in the truly distributed context.
    12. Given a solution to the distributed mutual exclusion problem, we can construct semaphores in a distributed system, and vice versa.
    13. In SR, what is the effect of sending a message to an op implemented as a procedure?
    14. Explain, with an example, the map/reduce idea behind Hadoop.
    15. We wish to maximize (a) concurrency, we prefer a (b) symmetric solution, we rate  (c) correctness much higher than efficiency, and cannot tolerate (d) deadlocks or (e) livelocks.  Defend why for each of these five.
  2. (20+5 points) Given a large (in the zillions) set S of integers, construct a bag B from it by duplicating every integer d times.  Present  the solution in C/Java+Linda (20 points), with  full explanation (5 points) written separately from the code. Assume that the tuple space already contains the set of integers <"S", x>, the size of this set <"SZ", sz>, and <"d", d> the number of duplications, <"p", p> the number of worker processes. The tuple space at the end should contain <"B", b>. You lose 5 points for each use of inp or rdp.
