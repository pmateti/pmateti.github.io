CEG 730 Distributed Computing Principles

Summer 2002// Final Exam // 75 minutes // 100 points
Prabhaker Mateti

This is a closed book, closed notes exam. Do not give or take help during the exam.
  1. (5*5 points) The following statements may or may not be (fully or partially) valid.  First, explain the technical terms occurring in the statement.  Then, explain/ discuss/ dispute the statement.  It is possible to write no more than, say, ten, sentences each, and yet receive full score.
    1. The property   ``Some day, if not at the end of CEG 730, you will appreciate and marvel at the book Concurrent Programming by Andrews.''  is neither a safety, nor a liveness property.
    2. The CSP code   [X!2 || Y!3]   is equivalent to
         [true -> X!2; Y!3 [] true -> Y!3; X!2]
    3. Every client/server program is based on RPC (-like) mechanism.
    4. Token-passing algorithms were discussed under the heading of ``asynchronous message passing'' because token-passing does not make sense under synchronous message passing.
    5. A typical machine (node) in a distributed system can run several concurrent processes. Each of these processes must maintain its own logical clock in order to correctly deal with the happened before relation.
  2. (5*5 points)
    1. Discuss why it is difficult to implement a semaphore in the truly distributed context.
    2. Describe the distributed implementation of semaphore of our text book.
    3. There are numerous ACK messages being sent.  Discuss if any or all of these are unnecessary.
    4. Does it implement a weak, strong (or neither) semaphore?
    5. Can any solution to the distributed mutual exclusion problem be thought of as a solution to the problem of implementing semaphores in a distributed system, and vice versa.
  3. (5*10 points) In a bag B of  b integers, it is known that each element appears exactly d times. We wish to delete the duplicates resulting in a set of only b/d distinct integers that occurred in the bag. It is guaranteed that d > 0, b > 0. Unfortunately, the value d is unavailable and needs to be computed by examining the bag B.   Two solutions, one in CSP and another in C-Linda are to be developed.  It is not required that both use the same "idea."  You are expected to present your design and implementation with full explanations.  As usual, we wish to maximize concurrency, we prefer a symmetric solution, and we rate correctness much higher than efficiency. 
    • (a) Design  and (b) implement a solution in C-Linda. Assume that the tuple space already contains <"B", xi> for all xi in B, and the size of the bag <"b", b>. You lose 5 points for each use of inp or rdp.
    • (c) Design and (d) implement a solution in CSP. Assume that the bag is already distributed, one element per process.  Thus, there are b processes, each with exactly one element.  After the algorithm terminates, some of these processes will be "empty handed."
    • (e) Defend why we wish to maximize concurrency,  why we prefer a symmetric solution, and  why we rate correctness much higher than efficiency.
