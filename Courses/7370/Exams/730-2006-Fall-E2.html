  1. (6*5 points) The following statements may or may not be (fully or partially) valid.   Explain/ discuss/ dispute the statement.  It is possible to write no more than, say, five thoughtful sentences each, and yet receive full score.

    1. Every distributed system must also be a concurrent system.
    2. Asynchronous  message passing is more fundamental than synchronous message passing.

    3. Token-passing algorithms were discussed under the heading of  asynchronous message passing because token-passing does not make sense under synchronous message passing.

    4. The happened before relation, as in the book, requires a logical clock for every process.  But  a typical machine (node) in a distributed system runs several processes. It is sufficient to maintain a logical clock per node that is shared by all processes running on that node.

    5. Peer-to-peer distributed computing is just an academic term for the commercial term client-server computing.
    6. The code V(s); P(s) is always equivalent to a no-op ("skip", or do-nothing).
       
  2. (5+5+5+5 points) It is possible to write no more than, say, five thoughtful sentences each, and yet receive full score.
    1. Discuss why it is difficult to implement a semaphore in the truly distributed context.
    2. Discuss if all of the ACKs are necessary in the solution discussed in the lectures.
    3. Discuss if any solution to the distributed mutual exclusion problem be thought of as a solution to the problem of implementing semaphores in a distributed system, and vice versa.
    4. Describe the problem of "termination detection."
       

  3. (10+20+20 points) You are given a large (in the zillions) bag B of pairs of unsigned integers (x,  d).  Each pair represents a horizontal line segment of length d in the Euclidean plane whose left end point is at (x, y) with y always equal to 0.   A segment (x1, d1) contains (x2, d2) if  x1 ≤ x2 and x1 + d1 ≥ x2 + d2.  We wish to compute the bag Bmin which minimizes the size (== number of pairs) of  B so that every original pair in B is contained in some pair in the resulting collection Bmin. 

    1. (5 + 5 points) As usual, we wish to maximize concurrency, we prefer a symmetric solution, we rate correctness much higher than efficiency, and cannot tolerate deadlocks or livelocks. Defend why we prefer a symmetric solution (5 points) , and  why we rate correctness much higher than efficiency (5 points) .

    2. (10 + 10 points) Fully explain (10 points) your solution design  in English/pseudo-code for Linda and on a fresh page implement  (10 points) it in C-Linda. Assume that the tuple space already contains <"t", x, d> of B, and the size of this bag <"sz", N>. You lose 2 points for each use of inp or rdp.

    3. (10 + 10 points)  Fully explain  (10 points) your solution design in English/pseudo-code for CSP.   It is not required that this use the same "design idea(s)" of the above. Implement  (10 points) on a fresh page the design in CSP.  The bag B is now generated by process Q whose structure is
      Q :: *[ ... --> P!(x, d) ].  Develop the process P and any others.
